VoxLog API

All endpoints require Authorization: Bearer {token} header (except /health).

POST/v1/voiceASR + dictionary + LLM polish + archive

The main endpoint. Send audio, get polished text back. Simultaneously archives to SQLite.

Request: multipart/form-data
audiofileWAV 16kHz mono, max 60s
sourcestringmacos | web | python_client
envstringhome | office
target_appstringCurrent foreground app name
Response: 
{
  "id": "uuid",
  "raw_text": "...",
  "polished_text": "...",
  "asr_provider": "qwen",
  "llm_provider": "openai_gpt",
  "polished": true,
  "duration_seconds": 3.5,
  "latency_ms": 1800,
  "target_app": "Claude Code",
  "env": "home",
  "created_at": "2026-04-01T10:00:00Z"
}
Errors: 401 (auth), 413 (too long), 422 (bad format), 502 (ASR failed) 
curl -X POST http://localhost:7890/v1/voice \
  -H "Authorization: Bearer token" \
  -F "audio=@recording.wav" \
  -F "source=macos" -F "env=home"
POST/v1/transcribeASR only, no polish

Transcribe audio without LLM polish or archiving. Useful for debugging ASR quality.

audiofileWAV 16kHz mono
envstringhome | office
POST/v1/polishLLM polish only, text in/out

Apply dictionary corrections + LLM polish to text. No audio, no archive.

textstringRaw text to polish
envstringhome | office
curl -X POST http://localhost:7890/v1/polish \
  -H "Authorization: Bearer token" \
  -d "text=嗯那个把首页pricing放到hero下面&env=home"

GET/v1/summaryAI-generated daily summary

Uses LLM to summarize the day's voice inputs into topics, decisions, and action items.

datestringYYYY-MM-DD (defaults to today)
GET/v1/statsUsage statistics + cost estimate

Returns recording count, duration, provider breakdown, avg latency, estimated monthly cost.

POST/v1/envSwitch network environment
envstringhome | office
